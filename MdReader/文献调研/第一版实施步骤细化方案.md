# AUV 单体控制算法仿真项目第一版实施步骤细化方案

> 本文档基于《AUV单体控制算法仿真项目方案_修改版.md》继续细化第一版任务。  
> 第一版目标不是追求复杂算法，而是把“水平面 3-DOF 欠驱动 AUV + LOS 路径跟随 + PID baseline + 海流扰动 + 参数偏差 + 数据展示”这一条基础闭环稳定做对。

---

## 1. 第一版总目标

第一版要完成一个可运行、可验证、可展示的最小闭环系统：

```text
参考路径
  ↓
LOS 制导
  ↓
期望航向 psi_d
  ↓
航向 PID
  ↓
偏航力矩 N
  ↓
纵向速度 PI/PID
  ↓
纵向推力 X
  ↓
3-DOF 欠驱动 AUV 模型
  ↓
实际轨迹
  ↓
误差与控制输入输出
```

第一版核心状态：

```text
state = [x, y, psi, u, v, r]
```

第一版控制输入：

```text
input = [X, N]
```

其中：

- `X`：纵向推力；
- `N`：偏航控制力矩；
- 横向力 `Y` 固定为 0，体现欠驱动 AUV。

---

## 2. 第一版不做的内容

为了避免第一版失控，以下内容暂不实现：

1. 完整 6-DOF 模型；
2. 垂向面深度控制；
3. 定点保持；
4. Backstepping；
5. SMC/NTSM；
6. ADRC/ESO；
7. MPC/NMPC；
8. ROS/Gazebo；
9. 复杂 Web 3D 动画；
10. 真实 AUV 几何建模和 CFD。

这些内容在第一版稳定后再逐步加入。

## 3. 第一版开发总流程

推荐按以下顺序推进：

```text
步骤 0：整理工程目录
步骤 1：建立 C++/CMake 基础工程
步骤 2：实现数学工具
步骤 3：实现日志输出
步骤 4：实现 PID 基础模块
步骤 5：实现单通道航向模型并验证 PID
步骤 6：实现 RK4 积分器
步骤 7：实现 3-DOF AUV 参数结构
步骤 8：实现无海流 3-DOF 动力学模型
步骤 9：实现纵向速度 PI/PID 控制
步骤 10：实现执行器饱和和 anti-windup
步骤 11：实现恒定海流和相对速度
步骤 12：实现直线路径 LOS
步骤 13：实现 LOS + 航向 PID 路径跟随
步骤 14：实现圆形路径和 S 形路径
步骤 15：实现 truth model / controller model 参数偏差
步骤 16：实现评价指标
步骤 17：实现 Python/Plotly 绘图脚本
步骤 18：整理第一版演示场景
```

每一步都应先完成验证流程并满足验收标准，再进入下一步。第一版开发时不要连续写很多模块后再统一调试，而应采用：

```text
实现一个小模块
  ↓
运行对应验证
  ↓
记录验证结果
  ↓
修正问题
  ↓
通过后进入下一步
```

如果某一步验证不通过，应停止后续开发，先把当前步骤的问题定位清楚。

## 6. 步骤 2：实现数学工具

### 目标

先把角度和坐标变换做对，避免后续仿真方向错误。

### 建议文件

```text
include/auv/math/angle_utils.h
include/auv/math/rotation_2d.h
src/math/angle_utils.cpp
src/math/rotation_2d.cpp
tests/test_angle_utils.cpp

```

### 必须实现

1. `wrapToPi(angle)`；
2. `deg2rad(deg)`；
3. `rad2deg(rad)`；
4. `rotationBodyToWorld(psi)`；
5. `rotationWorldToBody(psi)`。

### 验收标准

1. `wrapToPi(pi + 0.1)` 结果接近 `-pi + 0.1`；
2. `wrapToPi(-pi - 0.1)` 结果接近 `pi - 0.1`；
3. `psi = 0` 时，体坐标和惯性坐标方向一致；

### 验证流程

1. 编写或运行 `test_angle_utils`，测试 `wrapToPi()`、`deg2rad()`、`rad2deg()`；
2. 编写或运行 `test_rotation_2d`，分别测试 `psi = 0`、`pi/2`、`pi` 三种情况；
3. 打印几个典型向量变换结果，例如体坐标 `[1, 0]` 在不同航向角下对应的惯性坐标；
4. 检查旋转矩阵正反变换是否互为逆变换；
5. 如果任何一个符号方向不确定，先画草图或写小表格确认，不进入控制器开发。

## 8. 步骤 4：实现 PID 基础模块

### 目标

实现可复用的 PID 控制器，为速度和航向控制做准备。

### 建议文件

```text
include/auv/controller/pid_controller.h
src/controller/pid_controller.cpp
tests/test_pid.cpp
```

### 功能要求

PID 模块应支持：

1. `kp`、`ki`、`kd`；
2. 积分项限幅；
3. 输出限幅；
4. 重置积分；
5. 可选微分项滤波；
6. 可选 anti-windup。

### 验收标准

1. 输入固定误差时，比例项输出正确；
2. 积分项不会超过限制；
3. 输出不会超过饱和值；
4. 调用 reset 后积分项清零；
5. `dt <= 0` 时有保护逻辑。

### 验证流程

1. 运行 `test_pid`，先只开启比例项，确认 `output = kp * error`；
2. 开启积分项，持续输入固定误差，确认积分项增长但不超过积分限幅；
3. 设置很小的输出限幅，确认 PID 输出被正确截断；
4. 调用 `reset()` 后再次计算，确认积分状态和上一时刻误差被清空；
5. 使用 `dt = 0` 或负数做保护测试，确认程序不会除零或产生异常值。

### 通过后才能进入

步骤 5：实现单通道航向模型并验证 PID。

### 常见问题

| 问题 | 应对 |
|---|---|
| 积分风up | 加积分限幅和输出饱和反馈 |
| 微分项噪声大 | 第一版可先不用微分或加简单滤波 |
| 航向 PID 误差未 wrap | 航向控制器外部先计算 wrapped error |

## 9. 步骤 5：实现单通道航向模型并验证 PID

### 目标

在进入完整 3-DOF 前，先验证航向控制闭环。

### 简化模型

```math
I_z \dot{r} + d_r r = N
```

```math
\dot{\psi} = r
```

### 要做的事

1. 定义状态 `psi, r`；
2. 定义输入 `N`；
3. 使用航向 PID 跟踪 `psi_d`；
4. 加入 `N` 饱和；
5. 输出航向阶跃响应 CSV。

### 验收标准

1. `psi_d = 30 deg` 时，航向能逐渐收敛；
2. 航向误差不会在 `pi` 附近跳变；
3. 控制力矩不会超过限幅；
4. 不出现数值发散。

### 验证流程

1. 先不给控制器，给定一个恒定 `N`，确认 `r` 和 `psi` 的变化方向符合预期；
2. 接入航向 PID，设置 `psi_d = 30 deg`，输出航向阶跃响应 CSV；
3. 绘制 `psi`、`psi_d`、`e_psi`、`N` 随时间变化曲线；
4. 将初始航向设置在 `179 deg`，目标航向设置在 `-179 deg`，专门验证角度 wrap；
5. 如果航向朝反方向转，先检查误差符号和力矩符号，不进入 RK4 或 3-DOF 模型。

### 通过后才能进入

步骤 6：实现 RK4 积分器。

### 常见问题

| 问题 | 应对 |
|---|---|
| 航向朝反方向转 | 检查 `e_psi = wrapToPi(psi_d - psi)` 的符号 |
| 控制输入一直饱和 | 降低 PID 增益 |
| 航向振荡严重 | 减小 `kp`，增加适量 `kd` 或阻尼 |

## 10. 步骤 6：实现 RK4 积分器

### 目标

为后续动力学仿真提供稳定的数值积分方法。

### 建议文件

```text
include/auv/simulator/rk4_integrator.h
src/simulator/rk4_integrator.cpp
tests/test_rk4.cpp
```

### 功能要求

1. 支持固定步长；
2. 输入当前状态、控制输入和动力学函数；
3. 输出下一时刻状态；
4. 每次积分后对角度 `psi` 做 wrap。

### 验收标准

1. 对简单系统 `dx/dt = 1`，积分结果接近解析解；
2. 对简单指数衰减系统，误差在可接受范围；
3. 用 `dt = 0.01` 到 `0.02 s` 时结果稳定；
4. 积分后 `psi` 始终在 `[-pi, pi]`。

### 验证流程

1. 用 `dx/dt = 1` 从 `x=0` 积分到 `t=10s`，结果应接近 `10`；
2. 用 `dx/dt = -x` 从 `x=1` 积分到 `t=1s`，结果应接近 `exp(-1)`；
3. 对比不同步长 `dt = 0.1`、`0.02`、`0.01` 的误差变化；
4. 用只包含 `psi_dot = r` 的简单系统测试角度 wrap；
5. 如果积分误差异常，先检查 RK4 四个斜率的计算顺序。

### 通过后才能进入

## 12. 步骤 8：实现无海流 3-DOF 动力学模型

### 目标

完成第一版核心动力学模型。

### 状态定义

```cpp
struct State3DOF {
    double x;
    double y;
    double psi;
    double u;
    double v;
    double r;
};
```

### 控制输入定义

```cpp
struct ControlInput3DOF {
    double X;
    double N;
};
```

### 运动学

```math
\dot{x} = u\cos\psi - v\sin\psi
```

```math
\dot{y} = u\sin\psi + v\cos\psi
```

```math
\dot{\psi} = r
```

### 简化动力学

第一版可采用简化形式：

```math
m_{11}\dot{u} = X - d_u u - d_{uu}|u|u + m_{22}vr
```

```math
m_{22}\dot{v} = -d_v v - d_{vv}|v|v - m_{11}ur
```

```math
m_{33}\dot{r} = N - d_r r - d_{rr}|r|r + (m_{11}-m_{22})uv
```

如果调试困难，可以先忽略耦合项：

```math
m_{11}\dot{u} = X - d_u u - d_{uu}|u|u
```

```math
m_{22}\dot{v} = -d_v v - d_{vv}|v|v
```

```math
m_{33}\dot{r} = N - d_r r - d_{rr}|r|r
```

等基础闭环稳定后再打开耦合项。

### 验收标准

1. 无控制输入时，速度逐渐衰减；
2. 恒定正推力下，`u` 收敛到稳定速度；
3. 恒定偏航力矩下，`r` 收敛到稳定角速度；
4. 状态没有无故发散；
5. 单位和符号方向能解释。

### 验证流程

1. 设置初始 `u > 0`、`v = 0`、`r = 0`，控制输入为 0，确认 `u` 因阻尼逐渐衰减；
2. 设置初始速度为 0，给定恒定正推力 `X`，确认 `u` 逐渐上升并收敛；
3. 设置恒定偏航力矩 `N`，确认 `r` 的方向和 `psi` 的变化方向正确；
4. 分别关闭和开启耦合项，对比状态变化，确认耦合项不会导致异常发散；
5. 输出 `x-y` 轨迹和 `u-v-r` 曲线，确认运动趋势可解释。

### 通过后才能进入

步骤 9：实现纵向速度 PI/PID 控制。

### 常见问题

| 问题 | 应对 |
|---|---|
| 一加耦合项就发散 | 先关掉耦合项，基础稳定后再加入 |
| 阻尼符号写反 | 阻尼必须消耗能量，应阻碍当前速度 |
| 速度无限增长 | 检查二次阻尼和推力限幅 |

## 13. 步骤 9：实现纵向速度 PI/PID 控制

### 目标

让 AUV 保持一个期望纵向速度。

### 控制目标

```math
e_u = u_d - u
```

```math
X = K_{pu} e_u + K_{iu}\int e_u dt + K_{du}\dot{e}_u
```

第一版建议先使用 PI：

```math
X = K_{pu} e_u + K_{iu}\int e_u dt
```

### 验收标准

1. `u_d = 1.0 m/s` 时，`u` 能收敛；
2. `X` 不超过 `max_X`；
3. 积分项不会持续增大；
4. 加入阻尼后仍能维持期望速度。

### 验证流程

1. 在无偏航、无海流场景中设置 `u_d = 1.0 m/s`；
2. 绘制 `u`、`u_d`、`e_u` 和 `X` 曲线；
3. 将 `u_d` 改为 `0.5 m/s` 和 `1.5 m/s`，确认控制器仍能收敛；
4. 人为降低 `max_X`，确认输出饱和后积分项不会持续恶化；
5. 如果速度振荡明显，先调速度控制器，不进入路径跟随。

### 通过后才能进入

## 14. 步骤 10：实现执行器饱和和 anti-windup

### 目标

避免控制器输出超过物理限制，避免积分项失控。

### 建议文件

```text
include/auv/actuator/saturation.h
src/actuator/saturation.cpp
```

### 要做的事

1. 对 `X` 加入 `[-max_X, max_X]` 限幅；
2. 对 `N` 加入 `[-max_N, max_N]` 限幅；
3. PID 中加入积分限幅；
4. 如果输出已经饱和，限制积分继续朝饱和方向增长。

### 验收标准

1. 控制输入绝不超过限幅；
2. 大误差持续存在时，积分项不会无限增长；
3. 饱和解除后系统能恢复。

### 验证流程

1. 设置一个远超执行器能力的目标速度或目标航向，主动触发 `X` 和 `N` 饱和；
2. 检查日志中 `X_cmd` 和 `N_cmd` 是否始终在限幅范围内；
3. 记录 PID 积分项，确认饱和时积分项不会无限变大；
4. 将目标值恢复到可达范围，确认系统能从饱和状态恢复；
5. 如果恢复很慢或反向超调严重，优先调整 anti-windup 逻辑。

### 通过后才能进入

步骤 11：实现恒定海流和相对速度。

## 15. 步骤 11：实现恒定海流和相对速度

### 目标

正确模拟海流对 AUV 阻尼和轨迹的影响。

### 海流定义

惯性坐标系恒定海流：

```math
V_c^n = [V_{cx}, V_{cy}]^T
```

转换到体坐标系：

```math
V_c^b = R_n^b(\psi)V_c^n
```

相对速度：

```math
u_r = u - V_{cx}^b
```

```math
v_r = v - V_{cy}^b
```

阻尼使用相对速度：

```math
D(\nu_r)\nu_r
```

运动学中，位置变化仍使用地速：

```math
\dot{x} = u\cos\psi - v\sin\psi + V_{cx}
```

```math
\dot{y} = u\sin\psi + v\cos\psi + V_{cy}
```

### 验收标准

1. 海流为 0 时，结果与无海流一致；
2. AUV 停止推进时，会随海流漂移；
3. AUV 转向后，体坐标系中的相对海流方向会变化；
4. 横向海流会导致路径跟随横向误差变大。

### 验证流程

1. 设置 `V_c = [0, 0]`，确认结果和无海流版本一致；
2. 设置 AUV 无推进、无控制输入，给定 `V_c = [0.2, 0]`，确认位置沿惯性系 x 方向漂移；
3. 改变 AUV 航向角，打印体坐标系海流分量，确认它随 `psi` 正确变化；
4. 给定横向海流，运行直线前进场景，确认轨迹发生横向偏移；
5. 如果海流方向看起来跟着 AUV 转，先检查惯性系到体坐标系的旋转矩阵。

### 通过后才能进入

步骤 12：实现直线路径 LOS。

### 常见问题

| 问题 | 应对 |
|---|---|
| 海流方向随 AUV 一起转 | 海流应定义在惯性系 |
| 转向后阻尼异常 | 检查世界系到体系的旋转方向 |
| 位置没有漂移 | 运动学中需要加入惯性系海流速度 |

## 16. 步骤 12：实现直线路径 LOS

### 目标

将路径误差转换为期望航向角。

### 直线路径定义

一条路径段由两个点确定：

```text
P0 = [x0, y0]
P1 = [x1, y1]
```

路径切向角：

```math
\gamma_p = atan2(y_1-y_0, x_1-x_0)
```

横向误差：

```math
e_y = -(x-x_0)\sin\gamma_p + (y-y_0)\cos\gamma_p
```

LOS 期望航向：

```math
\psi_d = \gamma_p - atan2(e_y, \Delta)
```

其中 `Delta` 是前视距离。

### 验收标准

1. AUV 在路径左侧和右侧时，`psi_d` 修正方向正确；
2. `e_y = 0` 时，`psi_d = gamma_p`；
3. 前视距离 `Delta` 越大，修正越平缓；
4. `psi_d` 做角度 wrap。

### 验证流程

1. 先不接动力学模型，只对固定路径和多个测试点计算 `e_y` 与 `psi_d`；
2. 选取路径上方、下方、路径正上三个点，手动检查修正方向；
3. 分别设置 `Delta = 2`、`5`、`10`，确认前视距离越大，期望航向变化越平缓；
4. 使用接近 `pi` 和 `-pi` 的路径方向，专门验证 `psi_d` wrap；
5. 如果 AUV 后续会越修越远，优先回到这一步检查 `e_y` 符号。

### 通过后才能进入

步骤 13：实现 LOS + 航向 PID 路径跟随。

### 常见问题

| 问题 | 应对 |
|---|---|
| AUV 越修越远 | 检查 `e_y` 符号和 `atan2` 前的正负号 |
| 期望航向突变 | 对 `psi_d` 做连续化或限速 |
| 转弯太激烈 | 增大前视距离 `Delta` |

## 17. 步骤 13：实现 LOS + 航向 PID 路径跟随

### 目标

跑通第一版核心闭环。

### 闭环流程

```text
读取当前状态
  ↓
根据参考路径计算 cross-track error
  ↓
LOS 输出 psi_d
  ↓
航向 PID 计算 N
  ↓
速度 PI/PID 计算 X
  ↓
3-DOF 模型积分
  ↓
输出日志
```

### 验收场景

1. 直线路径，无海流；
2. 初始位置偏离路径；
3. 初始航向偏离路径；
4. 不同前视距离 `Delta`。

### 验收标准

1. 横向误差逐渐减小；
2. 航向误差逐渐减小；
3. AUV 不出现绕圈或发散；
4. 控制输入不长期饱和；
5. 实际轨迹能稳定贴近参考路径。

### 验证流程

1. 从最简单场景开始：直线路径、无海流、初始航向与路径方向一致，只设置横向位置偏差；
2. 绘制参考路径、实际路径和横向误差，确认误差收敛；
3. 再加入初始航向偏差，确认航向 PID 能修正；
4. 调整前视距离 `Delta`，观察收敛速度和平滑性；
5. 最后加入轻微海流，确认系统仍能保持基本路径跟随；
6. 如果出现绕圈、横向误差变大或长期饱和，不进入圆形和 S 形路径，先回查 LOS 符号、航向 PID 和速度控制。

## 18. 步骤 14：实现圆形路径和 S 形路径

### 目标

增加第一版展示效果。

### 圆形路径

可以用参数方程生成参考点：

```math
x_d = x_c + R\cos\omega t
```

```math
y_d = y_c + R\sin\omega t
```

第一版如果严格时间轨迹跟踪较难，可以把圆形路径离散成路径点，并使用最近点或路径段 LOS。

### S 形路径

可用：

```math
y = A\sin(kx)
```

然后生成一串路径点。

### 验收标准

1. 圆形路径不明显发散；
2. S 形路径航向变化连续；
3. 路径点切换没有频繁抖动；
4. 轨迹图能清晰展示参考路径和实际路径。

### 验证流程

1. 先生成圆形和 S 形路径点，不接控制器，单独画出参考路径；
2. 检查路径点顺序是否连续，是否存在重复点或过大间隔；
3. 接入路径跟随后，先降低期望速度运行，观察是否稳定；
4. 绘制 `psi_d` 曲线，确认没有不合理跳变；
5. 调整路径点间距和前视距离，直到轨迹平滑；
6. 如果路径点切换附近抖动明显，先增加切换半径或做期望航向限速。

## 19. 步骤 15：实现 truth model / controller model 参数偏差

### 目标

展示参数未知对控制效果的影响。

### 设计方式

仿真模型使用 truth 参数：

```text
truth_params
```

控制器或前馈补偿如果需要模型参数，则使用 nominal 参数：

```text
nominal_params
```

第一版 PID/LOS 对模型依赖较弱，因此参数偏差主要体现在仿真动力学响应上。

### 建议偏差

```text
mass_scale = 1.2
damping_scale = 1.3
thruster_scale = 0.9
```

### 验收标准

1. 无参数偏差和有参数偏差结果不同；
2. 参数偏差不会导致仿真立即发散；
3. 同一场景可重复运行并输出对比文件；
4. 文档中说明“这是给定范围内的鲁棒性能展示”，不表述为严格证明。

### 验证流程

1. 先运行 nominal 参数场景，保存为 baseline 结果；
2. 只修改一个参数类型，例如 `damping_scale = 1.3`，运行同一场景；
3. 对比轨迹、横向误差、控制输入，确认差异可解释；
4. 逐步加入 `mass_scale`、`thruster_scale`，每次只改变一个因素；
5. 如果仿真发散，先缩小偏差范围，不直接进入综合扰动展示；
6. 记录每组参数偏差，保证结果可以复现。

## 20. 步骤 16：实现评价指标

```text
mean speed error

```

2. 用仿真 CSV 计算指标，并和图中的误差峰值做人工核对；
3. 对同一场景重复运行两次，确认 summary 指标一致；
4. 对比无海流和有海流场景，确认指标能反映误差变大；
5. 如果指标和图中直观表现矛盾，先检查采样时间、单位和误差列。
